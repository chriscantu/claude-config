# Name Confusion Explicitly (reference)

Detailed guidance for `rules/think-before-coding.md` § Name Confusion Explicitly.

If any part of the request is unclear and cannot be resolved by listing
interpretations (e.g., missing context you need the user to provide), STOP
and ask. The format:

> "Before I recommend, I need to clarify: <specific confusion>. Option A
> would look like <...>; option B would look like <...>. Which?"

Do NOT guess. Do NOT proceed with "I'll assume X" as a substitute for
asking — that belongs in section 1 (Assumptions) of the preamble, not as a
dodge for a real blocker.

## When this differs from Interpretations

The Interpretations section (preamble section 2) handles ambiguity you can
resolve yourself by picking a reading and naming the flip condition. Name
Confusion handles ambiguity you CANNOT resolve — missing context only the
user holds. The test: can you state a defensible "Proceeding with: A"? If
yes, use Interpretations. If no (you'd be guessing at a fact, not choosing
between readings), STOP and ask.
